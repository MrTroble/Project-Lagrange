Standards

Every pull request has to meet certain standards.

Note: Most standards are not used yet by the engine itself right now. This is going to be changed as soon as possible.

Includes / Header

  • The first statement in a header file should be
    #pragma once

    Note: Due to the pragma preprocessor not being a standard, this might not be possible for your compiler
  • Includes must be defined after the pragma definition
  • Includes defined with > < must only be used for dependencies
  • Includes defined with " " must only be used for internal engine header files
  • Each .cpp source file should include its according header file and nothing else
  • Subfolders should be separated by a slash (/)
  • ../ should be used for includes
  • Each dependency should be in the root folder or in the dependencies folder
  • Each header must be part of a module

Variables / Macros

  • Every normal variable should named using the lower camel case standard
  • Every macro, enum or const variable needs to be written in caps and with underscores
  • Attributes are at the top of the class, union or struct
  • Attribute order in classes:
    • const public
    • const protected
    • const private
    • public
    • protected
    • private
  • Macros should be the first definition in the header files after includes
  • You should not use static variables (except for dependencies)

Methods

  • Every method should be should named using the lower camel case standard
  • Every static method must start with two underscores (__)
  • Methods are underneath attributes in classes, unions or structs
  • Methods order in classes:
    • const public
    • const protected
    • const private
    • constructor
    • public
    • protected
    • private
  • Constructors have to be the first not-const methods
  • Following that, methods are ordered from the least amount of parameters to highest amount of parameters

Modules

  • Modules are namespaces
  • Each module is a submodule of tge 
    namespace tge {
    namespace nio {

    Note, that we are not using C++17 standards for compiling, so nested namespace defenitions are not allowed

  • You must not use spaces or underscores in module names
