Standarts

Every pull request needs to meet certain standarts. And thus are as following.

Note: Currently most standarts are not used yet by the engine itself. This is going to be changed asap

Includes / Header

  • The first statement in a header file should be
    #pragma once

    Note: due to the pragma preprocessor not being a standart this might not be possible for your compiler
  • The second statements are the includes
  • Each .cpp source file should include it's according header file and nothing else
  • Subfolders should be seppareted by a slash (/)
  • ../ are allowed and must be used if you use dependencies
  • Each dependencie should be in the root folder or in the dependencies folder
  • Each header must be part of a moduel

Variables / Makros

  • Every normal variable should be snake case and lower case
  • Every makro or const variable needs to be written in caps and snake case
  • Attributes are at the top of the class or struct
  • Attribute order in classes:
    • const public
    • const protected
    • const private
    • public
    • protected
    • private
  • Makros should be the first definition in the header files after includes
  • You should not use static variables (except for dependencies)

Methodes

  • Every methode should be snake case and lower case
  • Every static methode must start with an underscore (_)
  • Methodes are underneath attributes in classes or structs
  • Methodes order in classes:
    • const public
    • const protected
    • const private
    • public
    • protected
    • private
  • Every methode should have a SINCE annotation (not needed if the class/struct/union already has one)

Modules

  • Modules are namespaces
  • Each moduel is a submodule of tge 
    namespace tge {
    namespace nio {

    Note that we are not using C++17 standarts for compiling so nested namespace defenitions are not allowed

  • You must not use spaces or underscores in module names

Class / Struct / Union

  • Every struct or union has the same standarts as classes
  • Every class should be written in upper camel case (pascale case)
  • Classes can have a SINCE annotation
  • Thus can contain methodes
